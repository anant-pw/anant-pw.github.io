AI Autonomous Testing Framework
Live

Autonomous bug detection & test case generation via LLM-guided browser crawl

A zero-script AI testing agent that crawls any web application, detects bugs intelligently, and auto-generates test cases — without a single manually written selector. The LLM decides what to explore next; Playwright executes it.

  • Auto-login detection — handles Cloudflare, multi-step login, SSO, cookie banners

  • Signal-gated bug detection — LLM only fires when real errors exist (console errors, failed XHR, DOM issues)

  • API testing layer captures all XHR/fetch calls during crawl and tests each endpoint
  • Self-healing actions with up to 5 fallback strategies when selectors shift

Sentinel QA — Human-in-the-Loop (HITL) AI Test Framework
Live

Requirements → approved test plans → AI execution → Jira triage — with a human in the loop

Sentinel takes unstructured requirements and turns them into executed, reported, and triaged test cases — end to end. A RequirementAnalyst agent (LangChain LCEL + SambaNova Llama 3.3 70B) converts requirements to JSON test plans, stores them via FastAPI + SQLite, and presents them in a Streamlit approval portal. Once approved, pytest + Playwright execute the plan. Failures trigger an AI BugReporter that opens Jira tickets with screenshots attached and appends analysis to Allure reports.

  • RequirementAnalyst agent converts plain-text requirements into structured JSON test plans via LangChain LCEL
  • Streamlit Human-in-the-Loop portal — humans review and approve plans before any test runs
  • FastAPI + SQLite backend stores plans, results, and AI memory across runs
  • LLM provider failover: SambaNova → Groq → Ollama (swap with one config line)
  • BugReporter agent triages failures with DB-backed pattern memory, opens Jira tickets + screenshots
  • Parallel execution via pytest-xdist; Allure reports auto-open with AI bug analysis attached per failure
  • ActionRegistry pattern replaces page-object models — Playwright actions driven by AI decisions
